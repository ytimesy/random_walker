# Random Walker

Cute random web wandering app built with Rails.

## Local development

```bash
bin/setup
bin/rails s
```

Open `http://localhost:3000`.

This app is now local-first:

- No billing or supporter checkout flow
- No server-side database
- Saved trails stay in the browser and can be exported as JSON

## Deploy to Render

This repo includes a [`render.yaml`](./render.yaml) for Render Blueprint deploys.

1. Push this repository to GitHub.
2. In Render, choose `New > Blueprint`.
3. Select this repository.
4. Set `RANDOM_WALKER_CONTACT_EMAIL` when prompted if you want a public contact link.
5. Deploy.

The web service uses Render's Ruby runtime, builds via [`bin/render-build`](./bin/render-build), starts via [`bin/render-start`](./bin/render-start), and exposes the Rails health endpoint at `/up`.

Notes:

- `SECRET_KEY_BASE` is generated by Render from the Blueprint.
- Production serves static assets when `RAILS_SERVE_STATIC_FILES=true`.
- The app does not require a server-side database.
- Saved trails are stored in each user's browser and are not synced to the server.
