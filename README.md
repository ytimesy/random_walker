# Random Walker

Cute random web wandering app built with Rails.

## Local development

```bash
bin/rails s
```

Open `http://localhost:3000`.

## Deploy to Render

This repo includes a [`render.yaml`](./render.yaml) for Render Blueprint deploys.

1. Push this repository to GitHub.
2. In Render, choose `New > Blueprint`.
3. Select this repository.
4. Set `RANDOM_WALKER_SUPPORT_EMAIL` when prompted.
5. Deploy.

The web service uses the existing `Dockerfile`, starts via [`bin/render-start`](./bin/render-start), and exposes the Rails health endpoint at `/up`.

Notes:

- `SECRET_KEY_BASE` is generated by Render from the Blueprint.
- Production serves static assets when `RAILS_SERVE_STATIC_FILES=true`.
- The app currently falls back to SQLite in production. Because this app does not persist user data yet, that is acceptable for an initial deploy. If you later add persistent data, move to Render Postgres or attach a persistent disk.
